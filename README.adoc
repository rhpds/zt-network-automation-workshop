=== Getting Started

* Create a git repo from this template
* Clone the repo and *cd* into it

=== Understanding the Basic Template Directory Structure

. Content Directory
+
.tree output
[source,shell]
----
.
├── README.adoc                 # Readme file
├── config                      # Defines VMs, network and firewall
├── lab-automation              # Ansible: containerlab up/down, optional router checks, Controller bootstrap
├── runtime-automation          # Scripts directory for setup/next/solve modules 
├── setup-automation            # Scripts directory when provisioning
├── ui-config.yml               # Configuration file
├── site.yml                    # Theme config
└── content                     # Content directory
    ├── antora.yml              # Antora configuration file. You can add "inline vars" here to render within your content
    └── modules/ROOT
        ├── assets
        │   └── images                       # Images used in your content 
        │       └── example-image.png
        ├── pages                            # Your content goes here
        │   ├── module-01.adoc               # First module of your lab, e.g. overview etc 
        │   ├── module-02.adoc               # Second module of your lab  
        │   └── module-03.adoc               # Third module
----

. Lab definition directory
+
.tree output
[source,shell]
----
config/
├── firewall.yaml                           # Define the egress/ingress rules to be allowed
├── instances.yaml                          # Define the instances to be created
└── networks.yml                            # Define the networks to be created
----


. Runtime Scripts Directory Structure
+
.tree output
[source,shell]
----
runtime-automation/
├── module-01                           # Directory name matches the content module (e.g. module-01)
│   ├── setup-control.sh                # Optional per-VM scripts; name must match `virtualmachines[].name`
│   ├── setup-containerlab.sh           # in config/instances.yaml (e.g. control, containerlab)
│   ├── solve-control.sh
│   ├── solve-containerlab.sh
│   ├── validation-control.sh
│   └── validation-containerlab.sh
├── ansible.cfg                         # Ansible configuration used with module playbooks
└── main.yml                            # Playbook to be called in each step
----

. Setup Scripts Directory Structure
+
.tree output
[source,shell]
----
setup-automation/
├── setup-control.sh                    # Executed on VM `control` when provisioned (see instances.yaml)
├── setup-containerlab.sh             # Executed on VM `containerlab` when provisioned
├── ansible.cfg                         # Ansible configuration used with module playbooks
└── main.yml                            # Playbook to be called in each step
----

=== Lab automation (containerlab + Controller API)

Playbooks under `lab-automation/` mirror the GitLab `network_workshop_containerlab` helpers, with fixes (correct `register`, no brittle `inspect` line slicing, configurable wait). They also include an optional *Automation controller* bootstrap using `awx.awx` (SCM project) and an optional `ansible.platform.token` task when Gateway env vars are set.

For a full walkthrough (inventory options, test order, router checks vs AAP, boot integration, troubleshooting), see link:lab-automation/README.md[lab-automation/README.md].

To bring up Containerlab **without Ansible** (manual CLI on the containerlab VM), see link:README-containerlab.md[README-containerlab.md].

* Run manually on the **control** node (after `ansible-galaxy collection install -r lab-automation/requirements.yml`):

[source,shell]
----
cd lab-automation
export CONTROLLER_PASSWORD='...'   # admin password; omit to skip Controller API steps only
ansible-playbook -i inventory/lab.yml playbooks/site.yml
----

* `setup-automation/setup-control.sh` runs `playbooks/site.yml` automatically when it finds `lab-automation/playbooks/site.yml` under `/home/rhel/zt-network-automation-workshop`, `/home/rhel/workshop`, or `$WORKSHOP_REPO_ROOT`.

* Inventory defaults: SSH to **containerlab** as **lab-user**; Controller at **https://127.0.0.1**. Override with `-e` or edit `inventory/` and `inventory/group_vars/`.

* Router smoke test (optional): `ansible-playbook -i inventory/lab.yml playbooks/gather_facts.yml -e containerlab_fip=<LoadBalancer IP>` (requires network collections).

=== Development Cycle

. Edit your content in `content/modules/ROOT/pages/`
. Use `git` to branch and commit your work
. Push your work to your repo
.. You should use `git tags` or `git branches` in production
.. However development items default to the head of `main`
